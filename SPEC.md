# UPH 工厂流水线计算器 - SPEC.md

## 1. Concept & Vision

一款专为工厂生产线工程师设计的节拍计算工具。输入每个工站的处理时间和缓冲区容量，模拟产品在流水线上的真实流动，找出瓶颈、计算总完工时间、输出 UPH（片/小时）。界面风格是工业控制台 + 数据仪表盘——沉稳、精准、高效，让工程师一眼看清产能。

- 文字主色：#e6edf3

**Motion**:
- 数据变化：数字滚动动画

- 模拟播放：实时推进时间线，120ms/秒（可调速）

## 4. Features & Interactions

### 4.1 工站配置
- 添加/删除/重排序工站（最少1个）
- 每个工站：名称（自定义）+ 处理时间（分钟）+ 缓冲区容量（≥1）
- 默认示例：涂层(6min, 缓冲2) / 切割(20min, 缓冲2) / 清洗(8min, 缓冲2)
- 输入校验：时间>0，缓冲≥1，名称非空

### 4.2 甘特图可视化
- 横轴：时间（分钟刻度，自动缩放）
- 纵轴：每片产品一行（P1, P2, P3...）
- 每个工站用不同颜色条表示处理时间段
- 空闲等待时段用灰色斜线标注
- 当前模拟时间有垂直红线
- 瓶颈工站闪烁标注
- 悬停显示：产品号、工站名、开始时间、结束时间

### 4.3 模拟引擎（核心算法）
事件驱动模拟：
1. 跟踪每个工站机器的可用时间
2. 跟踪每个工站缓冲区的当前产品数
3. 每当产品完成当前工站：检查下一工站缓冲是否有空位
   - 有空位 → 产品进入下一缓冲区，等待处理
   - 无空位 → 产品停留在当前工站输出端（不释放机器？）
4. 工站机器从缓冲区取下一个产品（按FIFO）
5. 关键约束：产品进入下一工站的"进入条件"= 该工站缓冲 < 容量上限
6. 特殊：产品完成最后工站 → 离开系统

**缓冲区语义**：
- 缓冲区存放"已完成上一工站，等待下一工站处理"的产品
- 机器空闲时，从自己的缓冲区取产品处理
- 当产品完成工站i时，检查工站i+1的缓冲是否满：
  - 未满 → 产品进入缓冲，释放工站i的机器
  - 已满 → 产品卡在工站i的出口，不释放机器（工站i暂停处理新产品）

### 4.4 结果指标
- 总完工时间（分钟 → 小时:分钟格式）
- UPH = 产品总数 / (总时间/60)
- 理论UPH（无限缓冲）：1 / 最慢工站时间
- 瓶颈工站（利用率最高的工站）
- 各工站利用率（处理时间 / 总时间）
- 产能利用率 = 理论UPH / 实际UPH（用于对比）

### 4.5 模拟控制
- 播放/暂停按钮
- 重置按钮
- 速度选择：0.5x / 1x / 2x / 5x（控制甘特图动画速度）
- 实时显示当前模拟时间

## 6. Technical Approach

- 纯 HTML + CSS + Vanilla JS（无框架依赖，单文件）
- 字体：Google Fonts（JetBrains Mono, Noto Sans SC）
- 甘特图：Canvas 或 CSS div 绘制
- 模拟状态：JavaScript 对象，requestAnimationFrame 驱动
- 数据存储：localStorage 保存用户配置的工站数据
